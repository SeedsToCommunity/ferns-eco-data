Scope
The Coefficients of Conservatism source exists today as three files in artifacts/api-server/src/services/coefficient/: data.ts (a static array of eleven C-value entries), metadata.ts (source constants and registry entry), and seed.ts (registry insert). Restructure it to the new pattern.
Target structure
The Source identifier is coefficient-of-conservatism. That identifier is used verbatim wherever the Source has a name in the code.

Create the Internal Data Providers workspace package at lib/internal-data-providers/ if it doesn't exist. Follow the pattern of lib/db/ and lib/api-envelope/ for package.json, tsconfig.json (composite: true), and the workspace registration. The package name is @workspace/internal-data-providers.
Move the data into the package at lib/internal-data-providers/src/coefficient-of-conservatism/. Inside:

data.ts — the COEFFICIENT_DATA array and the CoefficientEntry type. Not exported from the package.
index.ts — the Source Interface in code: query functions an Adapter calls. At minimum: getCoefficient(value: string): CoefficientEntry | undefined and listCoefficients(): CoefficientEntry[]. Only these query functions (and the CoefficientEntry type if needed for typing the Adapter) are exported.


Restructure the Adapter in artifacts/api-server/. The Adapter is whatever code calls the Provider and wraps its results in the EC envelope:

Route file at artifacts/api-server/src/routes/coefficient-of-conservatism.ts (rename from coefficient.ts if a route file exists; create if not).
Adapter services at artifacts/api-server/src/services/coefficient-of-conservatism/ (rename the existing coefficient/ directory). metadata.ts and seed.ts stay here. They are Adapter-side concerns (registry registration, metadata) and import the Source Interface from @workspace/internal-data-providers/coefficient-of-conservatism. They do not import data.ts from the Provider package; that file is not exported.
The route prefix on the EC Data Interface is /api/coefficient-of-conservatism/*. Rename from any previous prefix if it differs.


The Adapter imports the Provider only through its public Source Interface. No imports of the raw data array. If the package's exports are configured correctly, this is structurally enforced.
Update any callers elsewhere in the codebase that reference the old paths or names. The registry seed call in src/index.ts and any other wiring point need to match the new module locations and names.
Description fields stay as they are unless the existing text fails the playbook's Description Field Standards. If it does, fix it. technical_details should also include the deliberate scope exclusions clause (point 7a) — for Coefficients of Conservatism, the relevant scope statement is roughly "this Source defines the C-value methodology and scale only; per-species C-values are not included here." If the existing technical_details already says this, leave it.
