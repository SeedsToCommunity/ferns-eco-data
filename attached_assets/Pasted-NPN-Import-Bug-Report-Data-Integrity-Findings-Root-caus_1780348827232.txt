NPN Import — Bug Report & Data Integrity Findings
Root cause
The parseReportHtml function in artifacts/api-server/src/services/npn/import.ts assumed the Zope/MySQL report endpoint served moisture as a dedicated column at tds[6], separate from light. The live site did not — it combined light and moisture descriptors into a single tds[5] column. This caused a one-position left shift for every field from moisture onward, and a separate misalignment for notes vs range_michigan.
