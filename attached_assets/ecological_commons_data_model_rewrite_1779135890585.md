# Ecological Commons — Data Sharing Model

Ecological Commons is designed for builders.

The goal is not merely to publish ecological information for humans to read.  
The goal is to create interoperable ecological infrastructure that:
- applications,
- agents,
- organizations,
- researchers,
- stewardship groups,
- and communities

can build upon directly.

Ecological Commons is intentionally:
- federated,
- provenance-aware,
- source-preserving,
- and license-aware.

It is not intended to become a universal system of record.

---

# Why This Exists

Ecological restoration knowledge currently scales poorly.

Much of the best ecological knowledge exists as:
- PDFs,
- books,
- disconnected spreadsheets,
- static web pages,
- institutional memory,
- and expert mentorship.

These formats work for human reading, but they do not scale into:
- interoperable applications,
- ecological APIs,
- AI agents,
- stewardship tooling,
- or large-scale ecological coordination.

The result is repeated duplication:
- every group rebuilding species tables,
- every town recreating stewardship knowledge,
- every organization manually synthesizing ecological guidance again and again.

Ecological Commons exists to reduce that duplication cost while preserving:
- attribution,
- provenance,
- source identity,
- and trust.

---

# Commons Levels

Ecological Commons supports multiple commons levels.

Not all ecological datasets need the same openness model.

Different organizations may:
- want full public sharing,
- require attribution,
- or require reciprocal sharing of improvements.

Applications and agents should be able to filter datasets based on licensing compatibility and intended use.

---

# Supported Commons Levels

| License | Meaning | Commercial Use | Modification | Redistribution | Attribution Required | Share-Alike Required | Typical Use |
|---|---|---|---|---|---|---|---|
| CC0 | Public domain dedication | Yes | Yes | Yes | No | No | Foundational ecological infrastructure |
| CC-BY | Attribution required | Yes | Yes | Yes | Yes | No | Public ecological datasets with provenance |
| CC-BY-SA | Attribution + commons reciprocity | Yes | Yes | Yes | Yes | Yes | Community-curated ecological commons |

---

# CC0

CC0 datasets are intentionally released into the public domain.

These datasets maximize:
- interoperability,
- scalability,
- machine reuse,
- and ecosystem growth.

Typical examples:
- raw ecological observations,
- foundational species traits,
- interoperability schemas,
- public infrastructure layers.

CC0 is appropriate when the priority is:
> maximum reuse with minimum friction.

---

# CC-BY

CC-BY datasets require attribution.

Organizations retain visible provenance while allowing:
- applications,
- agents,
- APIs,
- and derivative systems

to build on the data.

This model supports:
- broad interoperability,
- visible source identity,
- and ecosystem growth.

CC-BY is likely the default model for many ecological datasets.

---

# CC-BY-SA

CC-BY-SA requires:
- attribution,
- and reciprocal sharing of derivatives.

If a system builds upon CC-BY-SA datasets, derivative datasets must remain open under the same license.

This model protects the commons from enclosure while still enabling:
- applications,
- APIs,
- agents,
- and commercial participation.

CC-BY-SA aligns strongly with:
- collaborative ecological infrastructure,
- community stewardship systems,
- and federated ecological knowledge networks.

---

# License-Aware Federation

Ecological Commons treats licensing as first-class metadata.

Applications and agents should be able to:
- include only CC0 datasets,
- require attribution-compatible sources,
- require ShareAlike compatibility,
- exclude incompatible licenses,
- or filter by organizational trust rules.

This allows:
- transparent governance,
- predictable interoperability,
- and scalable ecological infrastructure.

---

# Provenance Matters

Ecological Commons does not erase source identity.

Datasets should preserve:
- original organizations,
- source URLs,
- timestamps,
- attribution requirements,
- and licensing metadata.

Different ecological organizations may:
- disagree,
- classify differently,
- or prioritize different restoration goals.

Those differences are valuable.

The goal is not forced synthesis.

The goal is interoperable federation.

---

# Publicly Readable Is Not Enough

A downloadable PDF is not ecological infrastructure.

Ecological knowledge becomes scalable when it is:
- machine-readable,
- queryable,
- interoperable,
- attributable,
- and reusable by applications and agents.

Ecological Commons exists to help ecological knowledge move from:
- isolated documents,
toward:
- federated ecological infrastructure.
