Ann Arbor NPN — Consolidated state, last 10 days
What this source is
Greg Vaclavek's Michigan native plant database (nativeplant.com), a curated list of 130 Michigan native species with full ecological attributes — light, moisture, height, flowering time, growth form, conservatism/wetness/shade coefficients, notes, Michigan range, Cloudinary images (photographs and pen-and-ink drawings), and a per-species link back to Greg's original plant page. Greg's site is dormant; this is a static, never-reimport dataset.

The journey over the last 10 days
The source started life as a live PostgreSQL-backed scrape. The API server hit two DB tables (npn_species, npn_name_aliases) on every request. A scraper had imported Greg's data from nativeplant.com.

A column-shift bug corrupted the scraped data. The HTML report-page parser was merging the Light and Moisture table cells into one, which shifted every downstream field one slot left — height ended up in moisture, flowering time in height, and so on, for all species. The schema also used wrong column names (physiography, habitat) instead of the correct ecological field names.

The fix used Greg's own tab-separated text file as ground truth, bypassing the scraper entirely. All 130 species were corrected in the DB, the schema was fixed via direct DDL, and three species that the parser had missed entirely — CLEVIR (Clematis virginiana), HIEVEN (Hieracium venosum), and LIASCA (Liatris scariosa) — were inserted. Images for those three were also scraped directly from nativeplant.com and uploaded to Cloudinary. The corrected data was then committed as a static TypeScript file in lib/internal-data-providers/.

The routes were then fully converted to in-memory. The DB is no longer touched at all. All routes use sourceKind: "in-memory", sourceUrl: null, cacheStatus: "hit". The import endpoint (POST /api/ann-arbor-npn/import) was removed. The two DB tables were dropped. The knowledge_type was changed from "source_wrapper" to "dataset".

The source spec document was converted from a .docx file to a committed Markdown file (ANN_ARBOR_NPN_SOURCE_DOCUMENTATION.md), with Section 10 rewritten to describe the actual implemented system. The docx was deleted. That Markdown is served via the GET /api/ann-arbor-npn/documentation endpoint, wrapped in a standard FERNS JSON envelope (data.markdown).
